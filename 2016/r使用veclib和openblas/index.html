R使用vecLib和OpenBLAS

2016-10-30 #生活  

cd /Library/Frameworks/R.framework/Resources/lib
ln -sf /System/Library/Frameworks/Accelerate.framework/Frameworks/vecLib.framework/Versions/Current/libBLAS.dylib libRblas.dylib

在Mac OSX下，R可以轻松的调用Apple的vecLib或者openBLAS对矩阵运算进行提速。vecLib的调用非常简单，但是CRAN上给出的命令行是针对旧版的R，新的版本已不再适用(R 3.3.2)。 如果安装的R是从CRAN上下载的编译好的版本，那么只需要执行： 


cd /Library/Frameworks/R.framework/Resources/lib
ln -sf /System/Library/Frameworks/Accelerate.framework/Frameworks/vecLib.framework/Versions/Current/libBLAS.dylib libRblas.dylib
而让R用上openBLAS最简单的方法是从Homebrew安装（如果电脑上没有R）： 

brew tap homebrew/science
brew install r --with-openblas
或是从CRAN上安装R，再link到openBLAS: 

brew tap homebrew/science
brew install openblas
cd /Library/Frameworks/R.framework/Resources/lib
ln -sf /usr/local/Cellar/openblas/0.2.18_2/lib/libopenblas.dylib libRblas.dylib
如果出现运算结果不精确、不稳定情况而想要换回R默认的设置，执行以下命令： 

cd /Library/Frameworks/R.framework/Resources/lib
ln -sf libRblas.0.dylib libRblas.dylib
下面做一个简单的小例子以比较三者的速度。求解一个3000*3000随机矩阵的逆: 默认BLAS: 

> system.time(solve(matrix(rnorm(3e3^2), nrow=3e3)))
   user  system elapsed 
 52.128   0.241  52.570
Apple的vecLib: 

> system.time(solve(matrix(rnorm(3e3^2), nrow=3e3)))
   user  system elapsed 
  5.050   0.117   1.660
openBLAS: 

> system.time(solve(matrix(rnorm(3e3^2), nrow=3e3)))
   user  system elapsed 
  3.346   0.265   1.617
在这个小例子里，vecLib和openBLAS的速度几乎没有区别，都远远快于默认R的配置。如果怕麻烦，懒得下载编译，Mac OSX内置的vecLib就是很不错的选择。
